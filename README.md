## 📖 The Premise

> *Year 2047. Humanoid rabbits obsessed with disco music have taken over Earth. Humans are forced to dance endlessly under their glowing eyes. Those who stop... vanish.*

Once a year, the mysterious **Benefactor** offers an escape: the **Ceremony of Ascension**. One winner earns passage aboard the *New Esperanza* to Saturn.

Or do they?

**Choreomania: The Last Ascent** flips the classic "Wheel of Death" on its head. This isn't about surviving elimination — it's about being *chosen*. The Wheel of Hope spins, and salvation is just one raffle away.

---

## ✨ Features

| Feature | Description |
|---|---|
| **Inverted Mechanics** | Being picked means salvation, not death — a raffle you actually *want* to win |
| **Cinematic Narrative** | Synchronized voice-over, dramatic reveals, and a story that questions the meaning of freedom |
| **Full Sound Design** | Narration, ambient music, raffle effects, and an immersive audio landscape |
| **Smart Raffle Engine** | State-persistent selection system across 4 rounds: 16 → 8 → 4 → 2 → 1 |
| **Responsive Design** | Cinematic letterboxing optimized for desktop and mobile |
| **Tested Logic** | Jest-powered test suite ensuring reliability across all game states |

## 🛠️ Tech Stack

```
Frontend        HTML5 · CSS3 · Vanilla JavaScript
UX / Modals     SweetAlert2
Testing         Jest
Design          Figma
Deployment      Vercel
Version Control GitHub
```

**Architecture highlights:**
- Semantic HTML5 with native audio integration
- Custom CSS animations, letterboxing, and retro-futuristic styling
- Pure JS game logic with state management across raffle rounds
- Comprehensive unit and integration tests with Jest

## 🚀 Development Process

This project was built following **Agile methodology** across three focused sprints, tracked via [Trello](https://trello.com/b/2VTaF0sd/choreomania-the-last-ascent), combining solo development and mob programming sessions.

```
Sprint 1 ──── Conceptualization · Planning · Wireframing · Visual Design
Sprint 2 ──── Game Logic · Interaction Development · State Management
Sprint 3 ──── Testing · Responsive Optimization · Deployment
```

Regular standups, continuous integration, and iterative feedback loops kept the team aligned from concept to launch.
